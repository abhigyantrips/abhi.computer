I've seen Cloudflare automatically add a `SESSIONS` binding with a KV to my Astro site whenever I deploy. There isn't a specific way to stop it in the documentation, but [Unstorage](https://unstorage.unjs.io) (used by Astro under the hood) defines [a 'null' driver](https://unstorage.unjs.io/drivers/null) you can point to for disabling sessions.

```ts
import { defineConfig } from 'astro/config';

export default defineConfig({
    // ...

    session: {
		driver: {
			entrypoint: "unstorage/drivers/null",
		},
	},
})
```
